# 一、初识MyBatis

JDBC 是 Java 连接数据库的「底层标准」，MyBatis 是基于 JDBC 封装的「框架」，MyBatis 最终还是通过 JDBC 和数据库交互；

举一个例子：

原生jdbc 需要自己买菜、洗菜、切菜、炒菜、洗碗，每一步都要手动做，步骤繁琐但可控性最高

Mybatis 用预制菜（封装好的 JDBC 底层逻辑），只需自己写 “菜谱（SQL）”，其他步骤自动完成

所以，由于Mybatis和jdbc的底层是一样的，所以sql注入的这些漏洞其实是相通的。比如order by 无法预编译。在Mybatis也一样。

一、Mybatis如何与mysql进行交互

我们也可以将其分为以下几步

1、创建配置文件（mybatis-config.xml）

2、创建实体类（封装对象）

3、创建Mapper Xml文件（SQL语句）

4、修改配置文件（mybatis-config.xml）

5、main方法调用

6、释放资源

我们首先进行环境的配置，使用Maven导入mysql和Mybatis依赖

使用Maven创建项目时会有pom.xml文件，将我们的mysql和Mybatis依赖写入即可

<dependencies>
    <!-- MyBatis 核心依赖 -->
    <dependency>
        <groupId>org.mybatis</groupId>
        <artifactId>mybatis</artifactId>
        <version>3.5.14</version> <!-- 版本号可以调整 -->
    </dependency>

    <!-- 如果你要连接 MySQL，还需要 MySQL 驱动 -->
    <dependency>
        <groupId>mysql</groupId>
        <artifactId>mysql-connector-java</artifactId>
        <version>8.0.33</version>
    </dependency>
</dependencies>

# 1、创建配置文件（mybatis-config.xml）

我们需要在resources目录下创建一个mybatis-config.xml文件，里面放入我们的配置文件模板

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration "http://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
    <environments default="development">
        <environment id="development">
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
                <property name="url" value="jdbc:mysql://localhost:3306/mydb?useSSL=false&amp;serverTimezone=UTC"/>
                <property name="username" value="root"/>
                <property name="password" value="123456789"/>
            </dataSource>
        </environment>
    </environments>
</configuration>

# 2、我们创建实体类

由于我们的数据库中有id,username,password,level字段，所以我们的属性也就写四个

注意，实体类通常写在com.example.entity目录下

package com.example.entity;

public class User {
    private String id;
    private String username;
    private String password;
    private String level;

    public User() {
    }

    public User(String id, String username, String password, String level) {
        this.id = id;
        this.username = username;
        this.password = password;
        this.level = level;
    }

    public String getId() {
        return id;
    }

    public void setId(String id) {
        this.id = id;
    }

    public String getUsername() {
        return username;
    }

    public void setUsername(String username) {
        this.username = username;
    }

    public String getPassword() {
        return password;
    }

    public void setPassword(String password) {
        this.password = password;
    }

    public String getLevel() {
        return level;
    }

    public void setLevel(String level) {
        this.level = level;
    }
    @Override
    public String toString() {
        return "User{" +
                "id=" + id +
                ", username='" + username + '\'' +
                ", password='" + password + '\'' +
                ", level=" + level +
                '}';
    }
}

# 5、创建main方法实现查询

package com.example;

import com.example.entity.User;
import org.apache.ibatis.io.Resources;
import org.apache.ibatis.session.SqlSession;
import org.apache.ibatis.session.SqlSessionFactory;
import org.apache.ibatis.session.SqlSessionFactoryBuilder;

import java.io.IOException;
import java.io.Reader;
import java.util.List;

public class Main {
    public static void main(String[] args) throws IOException {
        Reader reader = Resources.getResourceAsReader("mybatis-config.xml");//读取配置
        SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(reader);//创建工厂
        SqlSession session = sqlSessionFactory.openSession();//建立连接
        String id = "1";
        List<User> users = session.selectList("UserMapper.getUserById",id);//执行语句
        session.close();//释放资源
    }
}

# 6、释放资源

虽然我们通过close也可以释放资源，但通常我们会使用try-with-resources来释放，更高效，并且异常时也会释放

public class Main {
    public static void main(String[] args) {
        Reader reader = null;
        SqlSessionFactory sqlSessionFactory = null;

        try {
            // 1. 读取配置并构建工厂 (工厂不需要在 try-with-resources 中)
            reader = Resources.getResourceAsReader("mybatis-config.xml");
            sqlSessionFactory = new SqlSessionFactoryBuilder().build(reader);
            try (SqlSession session = sqlSessionFactory.openSession()) {
                String id = "1";
                List<User> users = session.selectList("UserMapper.getUserById", id);
                users.forEach(user -> System.out.println(user.getUsername()));
            } // <--- 在这里 session 会自动关闭

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

这里只是简单的写一个流程，mybatis规范写法还是要看下手册。

# 二、Mybatis框架下的SQL注入

# （一）#与$

#与是该框架下两种拼接参数的写法，可以这么来说#相当于prepareStatement 而相当于Statement，这样就简单明了了。即#会对输入的“数据”进行预编译，而$却不会

我们来看一下案例

我们首先通过#来拼接参数看一下

<select id="getUserById" resultType="com.example.entity.User">
    SELECT * FROM users WHERE id = #{id}
</select>

我们传入String id = "1"

我们查看下控制台的打印日志

我们可以看到，它首先是将传入的参数位置使用？占位符预编译 然后再将1作为字符串传入，查询到数据并返回。

当我们改为$看下打印日志

# （三）in查询

SELECT * FROM users where username IN ('user1','a');

in查询会将in集合中符合条件的数据都查询出来，这里会有一个问题，即集合里的字段数量不确定。

如果使用

<select id="getUserById" resultType="com.example.entity.User">
    SELECT * FROM users WHERE  level In(#{level})
</select>

这一定是错的，它会将用户输入的，比如String = "1,2,3"

可能用户想的是 In(1,2,3)输入level等于1或2或3的用户信息；

但实际上程序会将1,2,3认为是一个值，即输出level=1,2,3的用户信息，显然这不符合我们的预想

所以，有些程序会直接拼接参数，这就会导致sql注入的出现

<select id="getUserById" resultType="com.example.entity.User">
    SELECT * FROM users WHERE  level In(${level})
</select>
String level = "1,2,3";
List<User> users = session.selectList("UserMapper.getUserById", level);
users.forEach(System.out::println);

==>  Preparing: SELECT * FROM users WHERE level In(1,2)
==> Parameters: 
<==    Columns: id, username, password, level
<==        Row: 2, user1, 123456, 1
<==        Row: 3, user2, 123456, 2
<==      Total: 2
User{id=2, username='user1', password='123456', level=1}
User{id=3, username='user2', password='123456', level=2}

此时虽然满足了业务需求，同时也出现了注入风险

String level = "1) or 1=1 #";
List<User> users = session.selectList("UserMapper.getUserById", level);
users.forEach(System.out::println);

==>  Preparing: SELECT * FROM users WHERE level In(1) or 1=1 #)
==> Parameters: 
<==    Columns: id, username, password, level
<==        Row: 1, admin, e10adc3949ba59abbe56e057f20f883e, 3
<==        Row: 2, user1, 123456, 1
<==        Row: 3, user2, 123456, 2
<==      Total: 3
User{id=1, username='admin', password='e10adc3949ba59abbe56e057f20f883e', level=3}
User{id=2, username='user1', password='123456', level=1}
User{id=3, username='user2', password='123456', level=2}

那么正确的方法该怎么写呢，Mybatis提供了正确的方法来实现，在SQL语句的文件中使用<foreach>

<select id="selectByIds" resultType="User">
    SELECT * FROM users
    WHERE id IN
    <foreach collection="list" item="id" open="(" separator="," close=")">
        #{id}
    </foreach>
</select>

并将用户输入类型转换为集合

List<Integer> ids = Arrays.asList(1, 2);
List<User> users = session.selectList("UserMapper.selectByIds", ids);
users.forEach(System.out::println);
